Project setup

We had to use GitLab for this project and we were put into random groups of 6. We were given 6 weeks to complete the project. We met our client, whose job was to travel to schools and talk about Chester University. He wanted a breakout puzzle game with a focus on computer science. Our client let us choose a theme that would be fun, we chose a pirate theme.

Implementation

We decided to use Razor Pages and .NET as we were more familiar with this rather than nodejs. We decided to use PixiJS after doing some research into 2D JS game engines. None of them has any experience whatsoever with web game making. I was given the role of a games programmer. I had to design educational games for a number of breakout levels. An example of this would be answer computer science questions and they had to complete a number of correct answers to move a ship towards some treasure whilst an enemy ship moved constantly towards the ship. Wrong answers took the players ship a bit backwards. This involved sprites which I got from the internet, but I told our asset designer to create assets for the game so it ended up looking much better. I also designed a basic binary game where the user had to translate the binary into letters that made up a word which would be a clue to the overall puzzle. Our meetings with the client went well, he seemed to agree with our direction and didn't ask for any major changes. We had an issue with one of our team not doing any work, but another teammate picked up their slack and we didn't set him much work so it wouldn't set us back as much. An issue with PixiJS was that the main update function would update too fast and cause lag so we needed to set a lower FPS.
